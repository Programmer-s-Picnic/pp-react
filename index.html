Section 2

What React and Vite actually are

Concepts
⚛️

What is React?

React is a JavaScript library for building user interfaces. Instead of manually changing HTML again and again, we build small reusable units called components. A component can represent a title, a button, a card, a form, or even a complete page.

React lets us describe what the UI should look like for the current data. Then React updates the page when the data changes.

⚡

What is Vite?

Vite is the tool that helps us develop and build the React project. It gives us a very fast development server and also creates the production-ready files for deployment.

Think of Vite as the workshop around React. React is the UI library. Vite is the tool that runs the workshop efficiently.

A simple way to remember this: React is what we build with. Vite is what helps us run and package it.

HTML

Gives the structure of the page.

CSS

Gives the style, color, spacing, and layout.

JavaScript + React

Gives the page behavior and component logic.

Section 6

Create the Hello World app

Code

We will now replace the default starter code with a small clear component.

src/App.jsx
export default function App() {
  return (
    <main style={{ fontFamily: 'Arial, sans-serif', padding: '40px' }}>
      <h1>Hello World</h1>
      <p>Welcome to my first React + Vite app.</p>
      <p>This app will be deployed on GitHub Pages.</p>
    </main>
  )
}

What is this code doing?

This file exports a function named App. That function returns JSX. JSX looks like HTML, but it is written inside JavaScript.

React reads this and creates the page you see in the browser. The heading and paragraphs become real HTML in the browser.

src/main.jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)

What does main.jsx do?

It imports the App component and tells React to render it inside the HTML element whose id is root.

So App is the thing you wrote, and main.jsx is the thing that places it on the page.

Save the file while npm run dev is running. The browser usually updates immediately. This is one of the nice things about modern frontend tools.
Section 7

A little more understanding: component, props, state

Core Ideas

Component

A component is a reusable piece of the interface. App is a component.

Props

Props are values passed into a component, almost like inputs to a function.

State

State is data inside a component that can change over time and cause the UI to update.

Simple state example
import { useState } from 'react'

export default function App() {
  const [count, setCount] = useState(0)

  return (
    <main style={{ fontFamily: 'Arial, sans-serif', padding: '40px' }}>
      <h1>Hello World</h1>
      <p>You clicked the button {count} times.</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </main>
  )
}

This example is not required for deployment, but it shows the real power of React: the page changes when the state changes.

Section 9

Prepare the app for GitHub Pages

Pages

Since our repository name is pp-react, the site will be hosted inside a subfolder. That means the app path is not just the main domain. It is the main domain plus /pp-react/.

If you forget the base path, your CSS and JavaScript may fail to load after deployment. This is one of the biggest beginner mistakes.
vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: '/pp-react/',
})

That one line, base: '/pp-react/', tells Vite where the app will live after deployment.

Section 11

Deploy with GitHub Actions

Deploy

We will add a workflow file. This workflow tells GitHub what to do whenever code is pushed to the main branch.

.github/workflows/deploy.yml
name: Deploy React to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Build app
        run: npm run build

      - name: Setup Pages
        uses: actions/configure-pages@v5

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
Step A

GitHub downloads your code

The workflow starts and checks out your repository.

Step B

GitHub installs Node and your packages

It prepares the same kind of environment needed to build the project.

Step C

GitHub runs the build

The dist folder is created.

Step D

GitHub Pages publishes the result

Your site becomes available online.

Section 14

Common beginner mistakes and exact fixes

Debug

Mistake: command not found

This usually means Node.js was not installed or terminal was not restarted after installation.

Fix: install Node.js, restart terminal, then run node -v.

Mistake: site looks broken after deployment

This usually happens when the base path is wrong.

Fix: in vite.config.js use base: '/pp-react/'.

Mistake: GitHub Actions does not publish

The repository may still be using a branch source instead of workflow deployment.

Fix: go to Settings → Pages and choose GitHub Actions.

Mistake: changes do not appear online

Sometimes the build was not triggered because the code was not pushed.

Fix: run git add ., git commit -m "update", and git push.

Section 17

Frequently asked questions

FAQ
Do I need to understand React deeply before using Vite?

No. Vite is simply the tool around your project. You can begin with very basic React knowledge and still use Vite comfortably.

Why do we need npm install?

Because the project depends on packages such as React, React DOM, and Vite. npm install downloads them.

Why is the base path so important for GitHub Pages?

Because your app is not deployed at the root domain. It is deployed inside the repo path, so asset URLs must point there.

Can I keep improving the app after deployment?

Yes. That is the normal workflow. Edit locally, test locally, commit, and push. GitHub Actions can deploy the updated version.

What should I learn next after Hello World?

Learn JSX, components, props, useState, event handling, lists, forms, routing, and fetching data.
